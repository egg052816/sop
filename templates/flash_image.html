XTS TEST – FLASH SW FLOW
整體原則 / 版本差異
General Rules

1. 除了 STS 外，所有 XTS 測試項目（CTS / CTS Verifier / GTS / GTS-Interactive / GTS-Root / CTS-on-GSI / VTS…）以及 Pretest，一律先燒錄 User Build SW。

2. STS 測試專用 Userdebug SW，不要混用在其他項目。

3. 本流程圖為 Android 15（A15）以上版本 使用：

  • A15 以上：需依圖燒錄 GTS-Root 進行測試。
  • A15 以下：不需要 燒錄 GTS-Root，其他流程相同。

CTS-on-GSI 燒錄流程
CTS-ON-GSI

CTS-on-GSI 需要額外從內網下載 Google GSI，並修改 fastboot bat 檔中的 system image 路徑，步驟如下：

  1. 開啟內網網站：http://10.57.42.97/XTS/ → 進入 VTS 資料夾。
    依照 JiRA 上標示的 Android 版本選擇，例如 Android 12.1：
    VTS-12.1_r14/ → 再進到底下的 R14 子資料夾。
  2. 在資料夾中找到對應架構的 signed GSI zip，例如：
    signed-gsi_arm64-img-13267304.zip
    下載並解壓縮後，裡面會有一個 system image，名稱類似：
    gsi_r11__system.img（通常會看到有兩個底線 __system.img）。
  3. 將這個 system image 複製到「客戶提供版本」的資料夾中（與原本 fastboot bat 同一層），
    找到對應的 bat 檔，例如：
    fastboot-flash-xts-gsi-cts-on-gsi.bat
  4. 使用文字編輯器打開該 bat 檔，搜尋 / 找到 flash system 相關指令，
    將原本的 system.img 檔名，改成剛剛複製過來的 __system.img，例如：
    fastboot flash system gsi_r11__system.img
  5. 儲存 bat 檔後，依照 SOP 執行該 bat 檔，完成 CTS-on-GSI 燒錄。
範例： fastboot-flash-xts-gsi-cts-on-gsi.bat 內原本可能是： fastboot flash system system.img 修改為： fastboot flash system gsi_r11__system.img
建議將實際使用的 GSI 版本與檔名記錄在 Waiver / 測試紀錄中，以便後續追蹤。
